persistence
/datum/controller/subsystem/persistence

! Unique String Persistence

this is the "generic store anything by key/value" system group specific string load is allowed but not required.

use if none of the others can work or if you're somehow too lazy to optimize your persistence system by making a new persistence type but somehow not lazy enough to optimize it into a string??

notice how we can't delete strings? that's intentional, and for optimization.
