code/__HELPERS/_logging.dm

/proc/log_admin Items with ADMINPRIVATE prefixed are stripped from public logs.
/proc/log_game All other items are public.
/proc/log_access_in <VStation_specific>
/proc/log_attack </VStation_specific>
/proc/log_world Log to both DD and the logfile.
/proc/log_runtime Log to the logfile only.
/proc/log_config Rarely gets called; just here in case the config breaks.
/proc/start_log For logging round startup.
/proc/log_tgui Appends a tgui-related log entry. All arguments are optional.
/proc/shutdown_logging Close open log handles. This should be called as late as possible, and no logging should hapen after.
/proc/key_name Helper procs for building detailed log lines
/proc/log_debug VSTATION SPECIFIC LOGGING.
/proc/log_error DEPRICATED. USE log_runtime(text) INSTEAD.
/proc/print_dir VSTATION LOGGING HELPERS Pretty print a direction bitflag, can be useful for debugging.
/proc/log_suspicious_login Writes to a special log file if the log_suspicious_login config flag is set, which is intended to contain all logins that failed under suspicious circumstances.
